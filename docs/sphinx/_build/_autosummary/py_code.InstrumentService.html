py_code.InstrumentService¶

Instrument Service

This script runs the instrument service for Karsa Tarkka TOF system.

InstrumentService connects to TofDaq Recorder data stream via KAcquisition. It (optionally) starts also KSignalProcessor instance, which processes the data in real time.

Running this script will start Pyro4 nameserver as a subprocess. Then it initializes KAcquisition instance, and thus requires TofDaq Recorder to be running. When KAcquisition (and KSignalProcessor) is running, it starts a Pyro Daemon to expose proxy objects for the threads. Finally, it tries to connect to the MainService via socket.io until succesful. Then the script enters main() function. After exiting main() it will execute shutdown().

Command line argument --ksp=False can be used to not run KSignalProcessor. For developing it is sometimes unnecessary and running without it is faster.

Bases: object

Remotely shared Lock object to synchronize file access

Can be used in place of a multiprocessing.Lock object. FileService holds Lock objects for each file in the current KDataPool. RemoteH5Lock objects can be used by other services to request file read/write access to files in the pool.

acquire(timeout=None)¶

Try to acquire remote lock

Emit 'file_write_request' with the set 'filename', 'timeout' and 'ns' to be forwarded via MainService to FileService. FileService will emit response 'write_lock_acquired' (via MainService) to the set namespace.

To properly handle the response, the emitting Client Namespace should implement:

def on_write_lock_acquired(self, data):

acquired = data.get('lock_acquired') if acquired:

remote_lock.lock.release()

Where remote_lock is the instance of this class to be acquired.

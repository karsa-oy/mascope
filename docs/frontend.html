Getting started

First Please check the following requirements are installed in your PC
    
        * Node version >=v12.13.1
        * Yarn (recommended) or npm 
        * git 
        
        Installation - 
        > open terminal then 
        > git clone https://bitbucket.org/bikram-karsa/karsa-electron.git
        > cd to karsa-electron
        > .\install.bat

Some Notes!!

Flow of Electron application - background.js->main.js->App.vue->MainUI.vue->other components

 

components in src/

  • MainUI.vue : This component is groups other components inside components directory as tab content. Subcomponets are imported and registered inside this vue component. 
  • MeasurementTab.vue: This component combines Datatab.vue (Deprecated component for processing offline files) and measurementtab component which does realtime processing of data acquired form tofdec recorder.
  • SeviceStatus.vue: This component displays the status of python services if they are runiing or not, Basically the idea is the backend services should be running before the UI can send the request to do some backend tasks. Therefore UI has to make sure all services in python are running. 
  • SpalshWindow.vue : This component displays a window which loads as soon as the application lauches from yarn. The ideas is that splash window makes sure the python services are loaded successfully and then only the UI launches. This component should make sure that python services are spawn by this vue component and then UI is loaded after. 

Project environment config

some of the environmental settins are configured in .env file which is copied from py_code directory to the root directory during the installation process using install.bat file. This environmental settings includes configurations needed to run server, towerk recorder simulation software path etc. The .env files is both used by electron and python as storing configurations. However original copy stays in python code. When shipping or building installer this is also necessary and has to be listed in vue.config.js file.

Note!! After you have run the project and made some changes in the .env file, the application has to be resarted to refelect the new changes otherwise changes will not be effective.

Electron configurations

The electron application configurations are located in three different files.

  1. packages.json - contains list of modules required by the node to create and run the project, some compilation/run/build process are configured here. 
  2. vue.config.js - This file contains extra configuration which is needed at the phase when installer is needed to create. Basically this file contains configurations that are used to copy extra directories and files which is necessary for electron to run. Eg. in our case when we make installer using yarn electron:build  it only copies files related to nodejs and vuejs however we might need extra files eg. python code directory, configs directory or some other assets directory. These are listed here. 
  3. src/background.js - This is main electron configuration and running instance where electron window related configurations are done. Eg. Either to hide or display default menu of electron, maximize or minimize windows, resize window or not, use node configuration or not etc. In short all the window related behaviour, window appreances style, tray icon style, Window event, parent child window etc. are defined here. 

Web sockets

As a means of communication between the UI and python websocket is used. Currently the socket.io websocket client is used. Since it is not a vuejs component or plugin, but a plain websocket client we need on handle websocket event in created property of vuejs otherwise the instances may not work at other properties eg. mounted, methods etc.

There are some advantages of using plaing websocket client without using component based module. Eg. simplicity, easy to create multiple websocket clients as necessary, you can write your own code handler in your custom way. 

For documentation it is recomended to look at socket.io  website's documentation page in client section.    

Compiling and Making installer file

Note: the current version of Plotly javascirpt has a bug after compilation is made. so to compile in right way, please follow the following steps.

Step 1

  • Open node_modules\plotly.js-dist\plotly.js
  • Find the script -

if (typeof define === "function" && define.amd) this.d3 = d3, define(d3); else if (typeof module === "object" && module.exports) module.exports = d3; else this.d3 = d3; }();

Then replace with

if (typeof define === "function" && define.amd) this.d3 = d3, define(d3); else if (typeof module === "object" && module.exports) module.exports = d3; else this.d3 = d3; }.apply(self);

Step 2

cd to the root directory of project and run > yarn electron:build

This will create a dist_electron directory if needed, else if it is already there there are auto generated files. It might take several minutes to complete. Once completed the directory will contain the following files.


                    ├── builder-effective-config.yaml
                    ├── bundled                                      (For web version, not needed in Karsa's case)
                    ├── karsa-desktop Setup 0.1.0.exe                (Full distributeable setup file, but has less friendly installation wizard)
                    ├── karsa-desktop Setup 0.1.0.exe.blockmap       
                    └── win-unpacked
(Contains compiled installation files which can be used as source to custom installer making software eg. simple-install maker software)

(Recommended Method)

Instead of using the desktop setup.exe file in the above list, Use of Simple installer software in the root directory is recommended.

The advantages are - It has much more flexibility in installation, small installation size, setting icons, more friendly installation procedure and uninstall feature and more options.

To use simple installer, open application InstallSimple-Portable\install simple.exe and go through the guided wizard.

When you reach a point where you need to select the source directory Then select -—> dist_electron\win-unpacked directory.

Now after successful compression the installer is created.



Note!!! Remember to remove .git files or will copy those as well, easiest is to cd to win-unpacked then remove all git files and then pack with installation maker software portable.
